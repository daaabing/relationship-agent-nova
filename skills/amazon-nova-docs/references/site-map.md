# Amazon Nova Documentation Map

## Primary Roots

- `https://docs.aws.amazon.com/nova/latest/userguide/` for Amazon Nova Version 1.
- `https://docs.aws.amazon.com/nova/latest/nova2-userguide/` for Amazon Nova 2.
- `https://docs.aws.amazon.com/nova-act/latest/userguide/` for Amazon Nova Act.

## Good Entry Pages

- `what-is-nova.html` for the Version 1 overview and model family summary.
- `what-is-nova-2.html` for the Nova 2 overview and current capability framing.
- `getting-started.html` for Version 1 onboarding and links into Bedrock console or API flows.
- `doc-history.html` for release history and changes that may affect "latest" answers.

## Common Topic Areas

- Model families, model IDs, and modality support.
- Getting started and Bedrock access patterns.
- Prompting, inference, and tool use.
- Grounding, reasoning, and code execution features.
- Fine-tuning or other customization paths on SageMaker.
- Region support, quotas, and release history.

## Frequent Pitfalls

- Do not mix Amazon Nova 1 and Amazon Nova 2 guidance without calling out the version explicitly.
- Do not assume pricing, region support, or model availability from memory. Re-check the live docs.
- Expect some operational details to live in linked Amazon Bedrock or SageMaker docs rather than in the Nova guide itself.
- Treat Amazon Nova Act as a separate service, even if the user refers to it loosely as "Nova".

- `site:docs.aws.amazon.com/nova/latest/nova2-userguide <topic>`

## Response Checklist

1. Confirm the version or product surface.
2. Open the matching AWS documentation page.
3. Re-verify if the user asks for the latest status or current support.
4. Cite the page title or URL in the answer.
